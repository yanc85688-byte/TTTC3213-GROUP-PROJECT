 Overview

This project builds an end-to-end **ETL pipeline** (Extract → Transform → Load) for **MR.DIY Malaysia** products in the **Home & Living (Household)** collection and performs basic analysis on:

- **Cheapest items** (lowest current price)
- **Highest discount %** items
- **Best deals by savings** (savings = original − current)

Academic purpose: **TTTC3213**.

## Data Schema (6 required attributes)

- `product_name`
- `current_price_rm`
- `original_price_rm`
- `discount_pct`
- `subcategory`
- `product_url`

During cleaning, additional fields are created:

- `discount_pct_raw` (raw text such as `Save 22%`)
- `savings_rm` (numeric savings when original price exists)

## Pipeline Summary

### A) Extract (Web Scraping)

- Scrapes public MR.DIY collection pages for **Household**
- Uses `product_url` as a unique key during scraping to reduce repeats
- Keeps price/discount values as raw text first (evidence of contamination)
- Adds a short delay (polite scraping)

### B) Transform (Cleaning)

- Normalize text fields
- Convert `discount_pct` from raw text → numeric
- Convert RM price strings → numeric floats
- Handle missing values and apply anomaly rules (swap original/current if needed)
- Compute `savings_rm`

### C) Load (Export)

Exports datasets for reporting and analysis.

## Output / Deliverables

Generated by the notebook:

- `mrdiy_home_living_raw.csv` (raw scraped dataset, ≥100 rows)
- `step3_dedup.csv` (intermediate checkpoint after dedup)
- `mrdiy_home_living_clean.csv` (final cleaned dataset)
- `before_after_contamination.png` (RAW vs CLEAN evidence chart)
